# Vue Test Task — Items Selection

Тестовое задание на позицию **Frontend-разработчика (верстка + Vue.js)**.  
Проект демонстрирует работу с состоянием, выбором элементов, компонентной архитектурой и аккуратной версткой.

## Описание задачи

Реализована рабочая страница со следующей логикой:

- **Нижний левый блок** — вещи пользователя  
  Можно выбрать **от 1 до 6 вещей**.  
  Выбранные элементы отображаются **слева сверху** в порядке выбора.

- **Нижний правый блок** — вещи на выбор  
  Можно выбрать **только одну вещь**.  
  Выбранный элемент отображается **справа сверху**.

Все элементы имеют одинаковую структуру (`id`, `name`), отличаются только данными.

Дизайн намеренно упрощён — основной фокус сделан на **логику, состояние и читаемость кода**.

## Стек технологий

- **Vue 3** (Composition API)
- **TypeScript**
- **Pinia** — управление состоянием
- **Vite** — сборка и dev-сервер
- **SCSS** — стилизация
- **ESLint + Prettier** — качество и форматирование кода

Дополнительно:
- Git (branches, PR)
- Chrome DevTools — отладка
- Проект подготовлен к контейнеризации (Docker — опционально)

## Архитектурные решения

- **Pinia** используется как единый источник истины для выбранных элементов
- Компоненты максимально простые и читаемые
- Типизация используется для данных и состояния

## Запуск проекта

### Локально

```bash
npm install
npm run dev
```

Проект будет доступен по адресу:
`http://localhost:5173`

### Сборка

```bash
npm run build
npm run preview
```
## Docker (опционально)

Docker добавлен как дополнительная возможность запуска и **не является обязательным**.

### Development-режим

Запуск проекта в dev-режиме (Vite dev server):

```bash
docker build -t vue-test-task .
docker run --rm -p 5173:5173 vue-test-task
```

После запуска приложение будет доступно по адресу:
`http://localhost:5173`

### Остановка контейнера

Контейнер останавливается стандартно через `Ctrl + C`.
